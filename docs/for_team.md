# Правила работы над проектом

## 1. Управление задачами
- Все задачи ведутся на привязанной к репозиторию доске ([GitHub Project Board](https://github.com/users/spooozy/projects/3)).
- Перед началом работы задача должна быть назначена на исполнителя и переведена в колонку **In Progress**.

## 2. Работа с ветками
- Название ветки создается на основе ID задачи и краткого описания (можно создать ветку прямо из описания задачи)
- Формат: `task-<id>-<short-description>`
- Пример: `task-1-setup-structure`

## 3. Коммиты (Conventional Commits)
- Текст коммитов — **только на английском**.
- Формат: `<type>: <description>` (см. Conventional Commits)
- Основные типы:
  - `feat:` (новый функционал)
  - `fix:` (исправление багов)
  - `docs:` (документация)
  - `assets:` (аудио, изображения)

## 4. Pull Requests (PR)
- Слияние с веткой `main` разрешено только через PR.
- **Обязательно одобрение как минимум одного участника.**
- В описании PR указывать связь с задачей: `Closes #<issue_id>`.
